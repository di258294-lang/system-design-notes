## 개요
이 저장소는 네트워크, 금융 시스템, 제도적 구조를 관찰하며 발견한 문제와 그에 대한 생각을 정리합니다.


[2026/02/08]
K-net은 한국의 국가융합전산망으로 알려져 있는데, 보통 우리가 아는 K-net은 www.hometax.com과 같은 전자정부 서비스일것입니다. 보통 k-'net'라고 하면 보통 .com부터 떠오르지 않으신가요?
그렇다면 전자정부 서비스도 아닌, 세종정부청사의 백본은 K-net일까요 아니면 gov-backbone일까요?
knet은 국가융합전산망인데, 그렇다고 하기엔 net이라고 하니까 .com부터 떠오르고, 국문으로 말하면 국가융합'전산망'입니다. 
그래서, '정부세종청사 백본망' 이라고 한다면 누구는 backbone(혹은 gov-backbone)이라고 생각하겠고 누구는 정부청사 백본이니 국가융합전산으로 생각해서 전자정부 서비스라고도 생각할 수 있겠군요
저는 정부세종청사의 백본을 K-NET이 아니라 gov-backbone라는 개념이 따로 생기는게 더 이해가 빠르게 될 것 같네요
